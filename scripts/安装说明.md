# cosmic_kb 安装与使用说明（拿到压缩包后看这个）

这是「苍穹历史项目本地理解工具」。纯本地运行，不上传你的源码。

> ⚠️ 本工具靠目录布局定位资产（`skill_assets/`、`comic-understand-long/` 必须是
> `cosmic_kb/` 的同级目录）。请**整包解压后原样使用**，不要只挑 `cosmic_kb/` 出来装。

## 1. 前置要求

- Windows + PowerShell（macOS/Linux 同理，命令改成对应 shell 即可）
- Python **3.10 或更高**：`python --version` 确认；没有就装 <https://www.python.org/downloads/>
  （安装时勾选 “Add Python to PATH”）

## 2. 安装（在解压出来的项目根目录里执行）

```powershell
# 1) 建并激活虚拟环境（强烈建议，避免污染全局）
python -m venv .venv
.venv\Scripts\Activate.ps1
#   若提示禁止运行脚本，先执行：
#   Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

# 2) 可编辑安装 + 常用可选组
pip install -e ".[parse,encoding,fuzzy,mcp]"
```

可选组按需取舍：

| 组 | 作用 | 建议 |
|----|------|------|
| `parse` | Java 静态分析（字段追踪旗舰功能的基础） | **必装** |
| `encoding` | GBK/混合编码自动探测（真实老项目几乎都需要） | **必装** |
| `fuzzy` | `ask` 自然语言中文模糊匹配 | 可选（不装会降级，能用） |
| `mcp` | 接 Claude / LLM 宿主的 MCP 服务器 | 仅需对接大模型时装 |

只想跑核心功能：`pip install -e ".[parse,encoding]"` 即可。

## 3. 验证安装

```powershell
cosmic_kb --version      # 打印版本号
cosmic_kb doctor         # 资产体检，应全部 OK（含 ok-cosmic-docs.db）
```

`doctor` 若报某项 MISSING，说明压缩包没解全或被改动过目录结构，请重新整包解压。

> 若 `cosmic_kb` 命令找不到，用等价写法：`python -m cosmic_kb.cli.main --version`

## 4. 快速上手（指向你要分析的苍穹老项目）

```powershell
# 建知识库（源码根 + 元数据 dym/cr/整包zip）—— 含 Java 字段级分析
cosmic_kb build "D:\path\to\项目源码根" "D:\path\to\元数据.zip"

# 旗舰：按层级精确定位字段 → 谁读/写它、事件函数、是否落库、源码行号
cosmic_kb trace "单据标识.字段标识"

# 单据钻取：操作集 / 插件 / 字段触达 / 风险
cosmic_kb bill "单据标识"

# 自然语言提问（确定性取证，不调大模型）
cosmic_kb ask "xxx 字段是谁改的"

# 信任优先：覆盖率 + 粗扫/高精度盲点对比
cosmic_kb coverage
cosmic_kb scan-compare

# 本地浏览器排障（输字段→表格→跳源码）
cosmic_kb web
```

## 5. （可选）跑自测确认环境无误

```powershell
pip install -e ".[parse,encoding,fuzzy,mcp,dev]"
pytest -q
```

## 6. 常见问题

- **`pip install -e .` 报 tree-sitter 编译失败**：升级 pip（`python -m pip install -U pip`）后重试；
  绝大多数平台有预编译 wheel，无需本地编译器。
- **激活虚拟环境被 PowerShell 拦**：`Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 后重开终端。
- **`doctor` 报 ok-cosmic-docs.db MISSING**：压缩包未解全，重新解压整包。
- **中文乱码**：终端用 UTF-8（`chcp 65001`）；工具本身已处理被分析源码的混合编码。

有问题把 `cosmic_kb doctor` 的完整输出发回给分发者即可定位。
