# cosmic_kb 安装与使用说明（拿到压缩包后看这个）

这是「苍穹历史项目本地理解工具」。纯本地运行，不上传你的源码。

> 本工具已是**自包含包**（语义文档、模板等随包），语义文档不再依赖同级目录。

**先确认你拿到的是哪种形态，两种装法不能混用：**

- **A. 一整个源码文件夹/压缩包**（解压后目录里能看到 `pyproject.toml`）—— 走「可编辑安装」
  路线，**整包解压后在项目根执行**下面第 2 节的 `pip install -e ".[...]"`。
- **B. 只有一个 `.whl` 文件**（单个文件，没有整个项目文件夹）—— **不要用 `-e`**（`-e` 只认
  本地源码目录，认不了打包好的 wheel），直接对着这个文件装：
  ```powershell
  pip install "cosmic_kb-<版本>-py3-none-any.whl[complete]"
  ```
  把 `cosmic_kb-<版本>-py3-none-any.whl` 换成你实际拿到的文件路径（相对/绝对路径都行），
  方括号里的可选组含义见下面第 2 节的表格。这种方式**不需要解压整包、不需要 `pyproject.toml`**，
  跳过本文档第 2 节，直接看第 3 节「验证安装」即可。

## 1. 前置要求

- Windows + PowerShell（macOS/Linux 同理，命令改成对应 shell 即可）
- Python **3.10 或更高**：`python --version` 确认；没有就装 <https://www.python.org/downloads/>
  （安装时勾选 “Add Python to PATH”）

## 2. 安装（场景 A：在解压出来的项目根目录里执行；场景 B 见上方提示，跳过本节）

```powershell
# 1) 建并激活虚拟环境（强烈建议，避免污染全局）
python -m venv .venv
.venv\Scripts\Activate.ps1
#   若提示禁止运行脚本，先执行：
#   Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

# 2) 可编辑安装 + 常用可选组（complete = parse+encoding+mcp+postgres 一次装齐）
pip install -e ".[complete]"
```

可选组按需取舍：

| 组 | 作用 | 建议 |
|----|------|------|
| `parse` | Java 静态分析（字段追踪旗舰功能的基础） | **必装** |
| `encoding` | 字符编码自动探测，个别文件不是 UTF-8 时的兜底（现在源码基本都是 UTF-8，用不上也不影响主流程） | 可选 |
| `postgres` | 直连苍穹底层库（只读）补原厂元数据 | 需补扩展单据原厂字段时装 |
| `mcp` | 接 Claude / LLM 宿主的 MCP 服务器 | 仅需对接大模型时装 |
| `complete` | 上面四组一次装齐（对话式安装口令固定用它） | **推荐** |

只想跑核心功能：`pip install -e ".[parse]"` 即可；想一次装全：`pip install -e ".[complete]"`。

## 3. 验证安装

```powershell
cosmic_kb --version      # 打印版本号
cosmic_kb doctor         # 资产体检：semantics/templates 应 OK（随包）
```

`doctor` 若报 semantics/templates **MISSING**，多为安装损坏，重装即可。

> 若 `cosmic_kb` 命令找不到，用等价写法：`python -m cosmic_kb.cli.main --version`

用户级隔离运行时装好固定版本，再跑 `cosmic_kb bootstrap` 一条龙建库、注册并校验四个 MCP 工具：

如果你已经在用**带终端/shell 权限的 AI 编程 agent**（Claude Code、Codex CLI、Cursor、
CodeBuddy CLI 等），下面「建 KB → 接 MCP」这一整条流水线**不需要你自己逐条敲命令**——打开
这个解压出来的项目文件夹，让 agent 读到这份说明，跟它说：

> 帮我按这份说明把 cosmic_kb 装起来：源码根是 `<你的苍穹项目源码路径>`，元数据文件夹是
> `<导出的 dym/cr/zip 所在文件夹>`。底层库只读账号信息我等下补，先帮我生成配置模板。
> 装完帮我把 MCP 也注册好。

它能替你跑完：建库（含下面「必须」的原厂元数据补齐）→ 把 MCP 配置写进这个 agent 自己认的
位置（Claude Code 是 `.mcp.json`、Codex 是 `~/.codex/config.toml`……）。**两处例外，agent
代劳不了**：

- **数据库口令**：不要把密码直接打在对话里（可能被记录）。让 agent 生成完配置模板后，自己
  在终端里设环境变量、跑一次连接测试确认连通，这一步手动做。
- **Qoder / Trae 这类只在图形化设置面板里粘贴 MCP JSON 的客户端**：粘贴进设置页这一步没有
  命令行入口，agent 没法替你点鼠标——按下方「把你的 AI agent 接到本工具」一节手动粘一次即可，
  这不是本工具的限制，是那类客户端本身没开放配置文件接口。

## 5. 快速上手（指向你要分析的苍穹老项目）

```powershell
# 建知识库（源码根 + 元数据 dym/cr/整包zip）—— 含 Java 字段级分析
cosmic_kb build "D:\path\to\项目源码根" "D:\path\to\元数据.zip"

# 旗舰：按层级精确定位字段 → 谁读/写它、事件函数、是否落库、源码行号
cosmic_kb trace "单据标识.字段标识"

# 单据钻取：操作集 / 插件 / 字段触达 / 风险
cosmic_kb bill "单据标识"

# 自然语言提问（确定性取证，不调大模型）
cosmic_kb ask "xxx 字段是谁改的"

# 信任优先：覆盖率 + 粗扫/高精度盲点对比
cosmic_kb coverage
cosmic_kb scan-compare

# 本地浏览器排障（输字段→表格→跳源码）
cosmic_kb web
```

### 必须：先直连底层库补原厂元数据（否则扩展单据会结构性半盲）

苍穹项目几乎都是"在原厂标准单据基础上做扩展"，ISV 导出的元数据包**只含扩展部分**，继承来的
原厂标准字段根本不在包里——跳过这一步，这类扩展单据的字段级排障会结构性半盲。**必须在第一次
`build` 之前配好**：找底层库的**只读**账号，让工具自动把原厂元数据拉回来补齐：

```powershell
cosmic_kb db-meta --init-config                      # 生成配置模板 cosmic_db.json，填 host/port/账号/schema
$env:COSMIC_DB_PASSWORD = "..."                       # 口令走环境变量，不写进配置文件
cosmic_kb db-meta --check --config cosmic_db.json     # 测试只读连接

cosmic_kb build "D:\path\to\项目源码根" "D:\path\to\元数据.zip" --db-config cosmic_db.json
```

连接**强制只读**，只取元数据表 XML，不落公网、结果照旧只进本机 KB。详见仓库
`docs/设计方案/扩展元数据识别方案.md`。

## 7. 常见问题

- **`pip install -e .` 报 tree-sitter 编译失败**：升级 pip（`python -m pip install -U pip`）后重试；
  绝大多数平台有预编译 wheel，无需本地编译器。
- **激活虚拟环境被 PowerShell 拦**：`Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 后重开终端。
- **中文乱码**：终端用 UTF-8（`chcp 65001`）。

有问题把 `cosmic_kb doctor` 的完整输出发回给分发者即可定位。

## 8. （可选）把你的 AI agent 接到本工具

装了 `mcp` 组后，本工具可作 MCP 服务器供任意 agent（Claude Code/Desktop、Cursor、Cline、
Windsurf、Codex…）调用取证。先 `cosmic_kb build` 出 KB，再在 agent 的 MCP 配置里加：

```json
{
  "mcpServers": {
    "cosmic_kb": {
      "command": "cosmic_kb-mcp",
      "env": { "COSMIC_KB_DB": "D:\\你的项目源码根\\cosmic_kb.db" }
    }
  }
}
```

Codex 用 `~/.codex/config.toml` 的 `[mcp_servers.cosmic_kb]`；Claude Code 在项目根直接认 `.mcp.json`。
接好后直接问「某字段是谁改的」「这张单有哪些插件」，agent 会自动调工具取证、带证据与苍穹语义作答。
详见仓库 README「接入任意 AI agent（MCP）」一节。

### 一键安装 CodeBuddy / Qoder / TRAE Skills

上面注册 MCP 已经足够——语义纪律（三态置信度、不臆造、入库判断）随 MCP `instructions` 自动带上，
任意 agent 不装额外东西也能"带语义"作答。CodeBuddy、Qoder、TRAE 用户可以安装包内的
`cosmic-kb-understand` 与 `cosmic-kb-setup`，获得固定的理解与初始化工作流：

```powershell
cosmic_kb skill install
# 未检测到宿主时显式指定
cosmic_kb skill install --agent all
# 检查是否缺失或过期
cosmic_kb skill status --agent all
# 卸载
cosmic_kb skill uninstall --agent all
```

CodeBuddy/Qoder 会直接安装到其标准 Skill 目录；TRAE 会生成导入包并打印设置界面的导入步骤。
卸载 TRAE Skill 时还需按命令提示在设置界面删除已导入项。安装后重启 Agent 或新开会话。
这些命令只管理 Skill，不自动注册或删除 MCP 配置。
