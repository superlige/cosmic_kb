# cosmic_kb V0.2.2 发版说明

发布日期：2026-07-23

V0.2.2 新增 Oracle 苍穹底层库只读元数据源，并把项目发版收敛成可失败续跑的一键脚本。

## 主要变化

- 新增 `OracleDriver`，基于 python-oracledb thin 模式连接 Oracle，无需安装 Instant Client。
- 新增 `SqlDialect` / `OracleDialect` 方言层，统一处理 PostgreSQL 与 Oracle 的占位符、
  批量成员判定、空串语义和当前时间查询差异。
- Oracle 批量查询按单个 `IN` 列表最多 1000 项自动分块；PostgreSQL 继续使用
  `= ANY(%s)` 单数组参数，原路径保持不变。
- 新增 `oracle` 可选依赖组。PostgreSQL 用户仍安装 `cosmic-kb[complete]`；Oracle 用户安装
  `cosmic-kb[complete,oracle]`，并在 `cosmic_db.json` 中设置 `driver=oracle`。
- README、安装说明和建库手册补充 Oracle 配置、升级方式及数据库字段含义。
- 新增 `scripts/release.ps1` 一键发版入口，自动完成版本与口令同步、一次全量测试、
  提交推送、构建、PyPI 摘要核对和 tag；失败后可用同一命令续跑。
- 补充苍穹开发排障 Agent 规划与项目精读文档，不改变现有 MCP 五工具接口。

## 升级注意

本版本沿用 KB schema **v20**，已有 PostgreSQL 用户升级后无需强制重建 KB。

使用 Oracle 时：

1. 安装 `cosmic-kb[complete,oracle]==0.2.2`。
2. 配置 `driver` 为 `oracle`、端口通常为 `1521`；`database` 填 service 名，
   `schema` 填元数据表 owner（通常为大写用户名）。
3. 数据库口令继续通过 `COSMIC_DB_PASSWORD` 或隐藏终端输入提供，不写入配置文件。

## 验证边界

- 自动化测试覆盖 Oracle/ PostgreSQL 方言、驱动注册、占位符、批量分块和 reader SQL 形状。
- 本次未连接真实 Oracle 实例；不同客户环境的 service 名、owner、账号权限与网络可达性仍需在
  实际环境执行 `cosmic_kb db-meta --ping` 验证。
- 如果 MCP 已注册，升级完成后需要重启或重连 agent，使新版本代码生效。
