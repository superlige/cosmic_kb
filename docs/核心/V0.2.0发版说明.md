# cosmic_kb V0.2.0 发版说明

发布日期：2026-07-17

V0.2.0 将 Java 调用分析从名字匹配为主升级为编译期符号绑定优先，并把跨类调用链、
方法反查和插件入口可达性纳入 CLI 与 MCP 的确定性取证能力。

## 主要变化

- 新增编译期符号解析层：自动发现 IDEA/Gradle 类路径，随包提供 `symsolver.jar`；无 Java
  环境时软降级，不影响基础建库。
- 新增完整 `call_edge` 调用边持久化与 `callers` 工具，可反查普通调用、方法引用及解析来源。
- `callers` 新增插件事件入口回溯和注册/启用状态核验，用五值 verdict 表达可达性与证据边界，
  弱证据不会被包装成死代码结论。
- `trace(kind="operation")` 新增程序化操作触发链，补齐跨类调用和方法引用触发场景。
- `trace`、字段数据流、落库点与操作触发优先使用符号层确定声明类，名字匹配仅作降级兜底。
- `build` 增加全流程进度反馈；MCP 工具面扩展为
  `trace` / `bill` / `resolve_fields` / `callers` / `cosmic_semantics` 五工具。

详细能力与证据边界见[阶段 12 工具能力提升总结](阶段12工具能力提升总结.md)。

## 升级注意

本版本将 KB schema 从 V0.1.6 的 **v16** 升级到 **v20**。旧 KB 不会自动迁移；升级包后必须
重跑一次 `cosmic_kb build`，或让 agent 执行一次 `cosmic_kb bootstrap apply`，完成 KB 重建后
再使用 `trace`、`callers` 等查询。

如果 MCP 已注册，升级完成后还需要重启或重连 agent，使新版本工具定义生效。
