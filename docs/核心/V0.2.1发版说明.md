# cosmic_kb V0.2.1 发版说明

发布日期：2026-07-20

V0.2.1 是操作触发链识别的准确性修复版本：移除苍穹平台并不存在的 `execOperate` 方法识别，
避免把项目内自定义的同名方法误判为平台操作触发或持久化调用。

## 主要变化

- `op_trigger` 仅识别平台标准的 `OperationServiceHelper.executeOperate` 和视图
  `invokeOperation` 调用，不再把 `execOperate` 当作同签名别名。
- 持久化检测与原厂标准单据自动发现同步移除 `execOperate`，三处识别口径保持一致。
- 更新操作触发链验收用例，并新增《操作触发链原理说明》，明确支持写法、参数解析、
  `unknown`/`dynamic` 边界和静态分析无法覆盖的场景。
- 补充苍穹开发排障 Agent 讨论规划，不影响 `cosmic_kb` 现有运行方式。

## 升级注意

本版本沿用 KB schema **v20**，无需因升级到 V0.2.1 强制重建已有 KB。若希望已有 KB 去除旧版本
可能写入的 `execOperate` 误报记录，请重跑一次 `cosmic_kb build`，或让 agent 执行
`cosmic_kb bootstrap apply` 重建 KB。

如果 MCP 已注册，升级完成后需要重启或重连 agent，使新版本代码生效。
