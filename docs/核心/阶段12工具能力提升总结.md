## 新覆盖的排障场景

**1. 方法引用触发链不再漏报（本阶段的起因，已闭环）。**
真实翻车：`trace(kind="operation")` 排查 `updatehistory` 触发链，漏了
`ContractRefundUpdateAssetOp` 经 `ContractService::updateRlateAssets`（方法引用语法）的一跳。
现在 `callers "cqkd.am.assets.service.contract.ContractService.updateRlateAssets"` 精确反查出
2 处调用（普通调用 + 当初漏掉的方法引用），均 expr 级解析、confidence=1.0。

**2. 「谁调了这个方法」有了确定性答案，agent 不再手写 Grep。**
以前验证一条调用链，agent 只能全项目正则搜——写严了漏方法引用，写松了把死代码误判成调用链。
现在 `cosmic_kb callers "Class.method"` 直接给出全部调用点（含行号、调用类型、解析来源、置信度）；
简单类名跨包重名时返回 `need_clarification` + 候选列表，不按调用数猜选。

**3. 死代码判定从「猜」变成「证」。**
`callers` 返回 0 结果时必附 `resolution_coverage`：仅当符号层 status=ok、无失败文件且
覆盖率 ≥95% 时才标为「查无调用方」的强证据（真实验收：测试类 `AjustAudiValidatorTest.main`
反查 0 且标强证据）；符号层降级或覆盖不足时明确提示「名字口径不足以断言死代码」。
反射、字符串配置、运行时动态分派仍是明示的边界，不冒充覆盖。

**4. 项目内同名方法/类彻底消歧。**
真实项目里 `getZqzdGroupKey` 一个名字在 4 个类各有一份定义，纯名字匹配必然混作一团。
编译期类型绑定（含 scope 兜底层）能把每个调用点钉到唯一声明类；平台 jar 里的同名类
（如项目内自定义 `SaveServiceHelper`）也按声明类 FQN 否决，不再误认。

**5. 落库判定（sink）与操作触发按声明类确认。**
`find_sinks`/`op_trigger` 从「方法名长得像」升级为「声明类 FQN 白名单确认/否决」，并新覆盖
静态导入裸 `save` 与 `SaveServiceHelper::save` 方法引用两种此前不可见的写法。

**6. jar/jdk 调用不再污染 unknown。**
以前解不出目标的跨类调用一律进 unknown 池；现在 43,037 条 jar 边 + 31,167 条 jdk 边被
确证为平台/JDK 方法，从 unknown 里剥离——unknown 池收缩为真正值得段二追查的残余。

**7. 精度来源逐条可审计（信任优先）。**
`field_access.edge_source`（local/symbol/mixed/heuristic）与
`operation_trigger.receiver_source`（symbol/text）落库并进入 trace/bill/coverage 返回；
`kb_meta.symbol_resolution` 保存 classpath attempts 完整诊断轨迹——「为什么这个 KB 没有
符号层」能从 KB 里直接读出来，呼应"处处置信度"哲学。

## 使用边界（诚实声明）

- **需要本机 java 8+ 运行时**（仅运行随包 `symsolver.jar`，不编译项目）。无 java 时
  `doctor` 提示「符号解析将降级」但不算体检失败；build 照常完成，全部边退回 tree-sitter
  名字匹配并标注 `heuristic`/`text` 来源——**无 classpath 不整体失能**。
- **类路径发现覆盖两代官方工程形态**（IDEA/.iml、金蝶官方 Gradle 模板）；裸源码导出等
  识别不出的情况用 `--classpath-dir` 显式指定 jar 目录兜底。
- **首次建库有一次性「冷盘税」**：jar farm 首读几十秒到几分钟（取决于磁盘），后续热盘 3–5 秒。
- **2.87% 的调用点仍解析失败**（failed 2,477，主要是 JavaParser 对泛型通配符/Stream 链
  方法引用推断的已知局限），这些站点带 `reason` 诚实落库；`callers` 对热点平台方法的 MCP
  返回按 28KB 预算分页，须翻至 `pagination.complete=true` 才是全量。
- **反射调用、字符串配置的类名、运行时动态分派**不在静态符号解析能力范围内，`callers`
  0 结果的强证据措辞已明示这三类边界。
