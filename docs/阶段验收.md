## 阶段 1 · 源码摄取 + 解析可信度报告 ✅

**背景**：接手陌生苍穹项目的第一个信任门槛——先证明能正确读取/解析"野生"源码
（混合编码、不可编译、多模块），用户才肯交出全量代码。

**目标**：递归摄取 `.java`（排除编译产物/VCS/IDE）+ 编码自动探测（含简体 CJK 偏置）
+ tree-sitter 容错解析（不因语法错误而崩）+ 覆盖率/可信度报告；命令 `cosmic_kb ingest`。

**验收结论**：✅ 通过。真实项目 `asset_management_sys` 全量三模块共 **1,345 个 Java 文件，
读取成功 100%、解析干净 100%、0 错误片段、无乱码**。

**相关命令**：
```powershell
pip install -e ".[parse,encoding,dev]"
cosmic_kb ingest "<项目源码根>"
cosmic_kb ingest "<路径>" --json | Out-File -Encoding utf8 coverage.json
```

## 阶段 2 · 元数据解析 + 整包/多包处理 ✅

**背景**：元数据（globalsdk 导出的 dym/zip）是 KB 的另一半来源。需把三类 dym
（单据/基础资料/动态表单）统一解析，并能批量吃下整包 zip；生产项目一个 zip ≈ 一个业务模块、
常多包并存，故还需多包跨模块汇总。

**目标**：
1. 三类 dym 统一为 `MetaModel`：字段不遗漏、层级正确（表头/分录/子分录）、类型可解析、
   下拉项带出、插件三类（界面/列表/操作）齐收。
2. 继承覆盖型操作的 hex oid → 标准操作，经继承根模板回填。
3. 整包双层 zip 解析，列出全部表单。
4. 多包 / 目录解析 + 跨模块（appKey）汇总。

**验收结论**：✅ 通过（含多包补强）。
- **单样例**：资产卡片 390 字段（表头 306 / 分录 75 / 子分录 9，无悬空）、插件 form4/list4/op17、
  4 个继承操作经模板译出；基础资料、动态表单分支均正确（动态表单字段解出且 DB 列为空）。
- **整包**：单包 315 dym **0 失败**；多包目录一条命令汇总 **2 包共 344 表单 0 失败**，
  按模块 `cqkd_assets×315 + cqkd_bd×29`。
- 测试 **36 passed**。
- 已知留接口（带入后续阶段）：反写插件无样例、占位未产出；动态表单继承根模板未提供、
  覆盖操作记 unknown；`.dymx` 多语言本阶段跳过。

**相关命令**：
```powershell
pip install -e ".[parse,encoding,dev]"
pytest -q                                          # 36 passed
cosmic_kb meta samples\bill\cqkd_assetcard.dym     # 单 dym
cosmic_kb meta samples\appzip\<包>.zip             # 单整包
cosmic_kb meta samples\appzip                      # 多包：目录下全部 zip 跨模块汇总
cosmic_kb meta samples\appzip --json | Out-File -Encoding utf8 multi.json
```
> 模板目录默认 `samples\bos_temp`，无需显式 `--template-dir`。

## 阶段 3 · 元数据 ↔ 代码桥接 ✅

**背景**：全项目最关键的桥——把元数据插件的 `<ClassName>` 全限定名定位到源码树里
真实的 `.java`。已拍板只认 ClassName 全限定名、不靠 ISV/前缀猜路径。

**目标**：
1. 以 `<ClassName>` 全限定名为唯一键，把 project 插件映射到源码文件。
2. 分类：精确命中 / 按类名降级命中 / 平台外部（无源码）/ 未找到 / 歧义；孤儿类收录。
3. 两套前缀（代码包前缀 / 元数据标识前缀）自动发现统计，仅作报告产物。
4. 孤儿类打轻量角色标签（常量类 / 真孤儿），常量类喂阶段 6/9、不进阶段 4 风险热点。

**验收结论**：✅ 通过。真实整包 `asset_management_sys`（源码 1345 文件 / 1349 顶层类型）
× `samples\appzip` 两包：
- **project 插件命中率 91.1%**（411/451；精确 404 + 按名降级 7），歧义 0。
- 平台外部 252（`kd.*` 与"末段全小写/含 `.kd.bos.` 的非 Java 类引用"均正确归外部）。
- 未找到 40（唯一 28 类，抽查确认源码确未给全，非桥接漏判）。
- **孤儿类 1075 = 常量类 207 + 真孤儿 868**；常量判定靠包名 `cons/const` 段（主，206/207）
  + 类名 `Const*` 强后缀（1），抽查零假阳后已收紧去掉 `Con/Cons` 短后缀。
- 双代码前缀 `cqkd / cqspb`，元数据标识前缀 `cqkd_`。
- 测试 **59 passed**（含桥接 23）。
- 已知留接口：常量类"符号→标识"字段值抽取按分工留至阶段 5/6（阶段 3 不解析字段体）。

**相关命令**：
```powershell
pip install -e ".[parse,encoding,dev]"
pytest -q                                                      # 59 passed
cosmic_kb bridge "<项目源码根>" samples\appzip                 # 整包桥接报告
cosmic_kb bridge "<项目源码根>" samples\bill\cqkd_assetcard.dym  # 单 dym 桥接
cosmic_kb bridge "<项目源码根>" samples\appzip --json | Out-File -Encoding utf8 bridge.json
```
