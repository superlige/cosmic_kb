## 阶段 2 · 元数据解析 + 整包/多包处理 ✅

**背景**：元数据（dym/zip）是 KB 的另一半来源；一个 zip ≈ 一个业务模块、常多包并存。

**目标**：三类 dym（单据/基础资料/动态表单）统一为 `MetaModel`；hex oid 经模板回填；
整包双层 zip；多包跨模块（appKey）汇总。

**验收结论**：✅ 通过。单样例资产卡片 390 字段（层级无悬空）；整包单包 315 dym 0 失败、
多包目录 2 包 344 表单 0 失败。

```powershell
cosmic_kb meta "<dym|cr 或整包 zip>"     # 单 dym / 整包 / 目录多包汇总
```

## 阶段 3 · 元数据 ↔ 代码桥接 ✅

**背景**：把元数据插件的 `<ClassName>` 全限定名定位到源码 `.java`（只认全限定名、不靠 ISV/前缀猜）。

**目标**：以 `<ClassName>` 为唯一键映射；五态分类 + 孤儿收录；双前缀自动发现；孤儿打角色标签。

**验收结论**：✅ 通过。真实整包 project 插件命中率 91.1%、歧义 0、孤儿 1075（常量 207 + 真孤儿 868）。

```powershell
cosmic_kb bridge "<项目源码根>" "<dym|zip|目录>"
```

## 阶段 5+6+7 · 字段级排障引擎（旗舰）✅

**背景**：产品方向从「项目普查」重定向为「字段级排障导航」（用户 2026-06-17 拍板）——消灭
苍穹排障痛点「把元数据里一堆插件全路径逐个复制到代码里翻」。

**目标**：输入字段标识 → 列出所有读/写它的**插件 + 事件函数 + 是否落库 + 行号 + 源码路径**，
按**实体坐标 (单据·层级·分录)** 分组精确定位。一次实现 阶段 5（事件/字段）+ 6（调用链，
类内 + 跨类回溯）+ 7（落库判定 = 事件相位 × 操作类型 × 调用链到 sink）。

**验收结论**：✅ 通过（经四轮返工至用户验收）。
- Java 语义层 `java/`（ast_index/constants/plugin_classifier/event_extractor/field_access/
  call_graph/persistence/project_graph/analyze），全 tree-sitter 容错、零硬依赖。
- 数据包来源识别覆盖：事件入参（单包/数组）、ORM load/loadSingle/query、转换插件目标/源单、
  getEntryEntity、for-each、lambda 行变量、stream 派生集合、DynamicObject 入参跨类传播。
- 查询入口按层级显式录入（点号段数定层级）；判不准进「可能命中」、来源判不出进「未定位」，不遗漏。
- 未绑定的苍穹插件（AbstractTask/WebApi/工作流）也作跨类入口回溯。
- KB schema **v7**（`field_access` 增 `access_class` + 坐标索引）；报告 `field_trace` + `bill_view`；Web 重做单页。
- 真实库复验 `cqkd_assetcollateral.cqkd_collateralstatus`：CollateralOp + CollateralService +
  AutoExecuteCollateralTask 写入全部精确命中、落库、带跨类调用链 + 源码行号。

```powershell
cosmic_kb trace cqkd_assetcard.cqkd_entry.cqkd_amount   # 点号精确定位
cosmic_kb bill cqkd_assetcard                            # 单据钻取
```

## 阶段 10 增补 · 方法出向调用导航 method_calls ✅

**背景**：原 `read_method`（方法级深读：复述方法源码 + 语义摘要）在交付人工测试时被否——段二
形态拍板为**大模型直接读本机源码 + 挂苍穹 skill**，复述源码 / 列平台·`equals`·常量调用 / 做自然
语言解释这些大模型自己做得更好，静态层在这块零增量甚至刷屏噪声（如 `super.propertyChanged()`、
`e.getProperty()`、常量类 `equals` 全被列成调用）。确定性扫描层对一个会读源码的大模型唯一不可
替代的，是**野生、多 ISV 前缀、不可编译码上的"跳转到定义"**。故把方法级能力收窄重置为纯调用导航。

**目标**：给定 类全限定名 + 方法名 → 只回该方法调用的**项目内**方法（调用名 + 目标类全限定名
`target_fqn` + 目标源文件 `target_relpath` + 调用行号），供大模型顺调用链逐层读源码下钻。**不回
源码全文**（大模型自读）、**不列平台/外部/落库 sink 调用**（噪声）、**不做字段落库取证**（归
`trace`）；接收者类型解不出 → 不收录（宁缺毋滥，不臆造）。schema 不变 v9。四入口：CLI `calls`
+ MCP 工具 `method_calls` + `ask` 意图 `method_calls` + `trace`/`bill` 跳转提示。

**验收结论**：⏳ 待人工测试。`pytest -q` **180 passed**（`tests/test_method_calls.py` 9：项目内调用
解析+噪声不收 / 跨类下钻 / 源码根自动解析 / 三类未命中候选 / resolver+builder+MCP 同口径）。诚实
边界：tree-sitter 未装或源码读不到 → 给空清单 + 明确提示、不臆造；同名类/重载列候选反问。

```powershell
cosmic_kb calls "<类全限定名>" "<方法名>"   # 该方法调了项目内哪些方法→目标类/源文件/行，供顺调用链读源码下钻
```
