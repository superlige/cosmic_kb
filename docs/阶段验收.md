## 阶段 0 · 脚手架 + 资产复用 ✅

### 实现了什么
- 建立 Python 包 `cosmic_kb/`，按两段式架构分出子包：
  `ingest / metadata / java / bridge / graph / semantic / context / report / cli`。
  每个子包的 `__init__.py` 写了职责说明与计划模块，**骨架自描述**。
- `cosmic_kb/_assets.py`：资产接线层，把现成的 `comic-understand-long/references/`、
  `skill_assets/ok-cosmic-docs.db` 定位进来，供后续 SDK 符号解释复用。
- `cosmic_kb/cli/main.py`：命令行入口，提供 `--version`、`doctor`（资产体检）、
  `plan`（列出规划中的子命令）。
- `pyproject.toml`：可编辑安装、`cosmic_kb` 脚本入口、可选依赖分组
  （`parse` / `encoding` / `fuzzy` / `dev`），阶段 0 运行期**零硬依赖**。
- `tests/test_smoke.py`：冒烟测试（版本可读、CLI 可跑、子包可导入、资产可定位）。

### 怎么验收

```powershell
cosmic_kb --version      # 有版本输出
cosmic_kb doctor         # 资产体检
pytest -q                # 冒烟测试通过
```

### 验收标准
- `pip install -e .` 成功；`cosmic_kb --version` 输出 `cosmic_kb 0.1.0`。

## 阶段 1 · 源码摄取 + 解析可信度报告 ✅

> 定位：**第一个信任门槛**。指向一坨"野生"目录，证明能正确读取与解析，
> 用户才会放心交出全量代码。

### 实现了什么

**1) 源码摄取 `ingest/scanner.py`**
- 递归发现 `.java`（`os.walk` topdown，就地裁剪子树）。
- 排除编译产物 / VCS / IDE / 依赖缓存：`target build out bin dist .git .svn .idea
  node_modules .gradle` 等。
- 符号链接默认不跟随（避免环路与越界），跳过记录单列。
- **编码自动探测**（中文老项目核心）：
  1. BOM 优先（utf-8-sig / utf-16）；
  2. 严格 UTF-8 解码成功 → utf-8；
  3. 借 charset-normalizer 猜；
  4. **CJK 偏置**：苍穹是大陆简体软件，charset-normalizer 在短样本上常把简体
     GBK 误判成 big5，故 gb18030 能严格解出时优先 gb18030（GBK 超集）；
  5. 兜底 gb18030 + 容错解码，宁可标低置信也不臆造。
- 产物：`SourceFile`（路径/相对路径/字节数/编码/置信度/文本/错误）、
  `ScanResult`（文件清单 + 跳过目录/链接）。

**2) Java 解析 `java/parser.py`**
- tree-sitter-java **惰性装载 + 缓存**；未装 `[parse]` extra 时优雅降级
  （状态标 `unavailable`，不崩）。
- 依赖 tree-sitter 的**错误恢复**治"不可编译"代码 —— 绝不因语法错误而崩。
- 逐文件产出**解析状态**：`ok`（干净）/ `partial`（含错误片段，已局部恢复）/
  `skipped`（无文本）/ `unavailable`（未装解析器）。
- 统计 ERROR / MISSING 节点数、总节点数、行数；记录**错误片段行号**
  （`error@Lxx` / `missing@Lxx`，最多 50 条防撑爆）。
- 仅做"能不能解析、干不干净"，**不抽语义**（类型/事件/字段留给阶段 5+）。

**3) 覆盖率 / 可信度报告 `report/parse_coverage.py`**
- 把摄取层与解析层串起来，逐文件合并成 `FileCoverage` / `CoverageReport`。
- 统计：扫描总数 / 读取成功率 / 解析干净率 / 编码分布 / 低置信编码清单 /
  含错误片段文件清单 / 跳过目录。
- 输出**信任结论**（干净率 ≥95% 判"可放心给全量代码"，≥80% 判"整体可信"，
  否则提示先排查）。
- 文本渲染（人看）+ `to_dict()` JSON（机器读 / 后续入库）双输出。

**4) CLI 命令**
```
cosmic_kb ingest <路径> [--json] [--follow-symlinks] [--max-error-files N]
```
- 退出码：正常 0；无 .java 或有读取失败为 1；路径不存在为 2（便于脚本判断门槛）。

**5) 测试 `tests/test_ingest.py`**
- 覆盖摄取/排除、编码探测（utf-8/BOM/GBK 中文）、解析（干净/损坏恢复/无文本）、
  覆盖率报告、CLI（文本/JSON/缺失路径）。
- 全量 **18 passed**。

### 怎么验收
```powershell
pip install -e ".[parse,encoding,dev]"
cosmic_kb ingest "<你的苍穹项目源码根>"
# 留档：cosmic_kb ingest "<路径>" --json | Out-File -Encoding utf8 coverage.json
```

### 验收标准
- 读取成功率接近 100%；解析干净率高（理想 ≥95%）。
- 中文文件无乱码（抽查"低置信编码"清单）。
- 错误片段定位准确（按行号对照源码确为语法异常，非误报）。
- `target/build/.git` 等被正确排除。

### 真实验收记录

- **结果**：

  | 指标 | 数值 |

  | 扫描文件 | 751 |
  | 读取成功 | 751（100.0%） |
  | 解析干净 | 751（100.0%） |
  | 含错误片段 | 0 |
  | 代码行数 | 127,929 |
  | 编码分布 | utf-8 × 751 |

- **结论**：✅ **通过**。读取/解析 100% 干净、无乱码、无错误片段。
  解析可信度获验证，可向工具交出更多模块 / 全量代码，进入阶段 2。
